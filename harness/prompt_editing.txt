## Working approach (checkpoint-first, then iterate to fidelity)
You have a soft budget of about 15 minutes — enough for several refinement passes. Use it; do not stop at the first valid solid.

1. Import the starting solid: import_cad_file("input.step", "part"). Inspect it — render_view() it and use measure() / find_holes / find_hole_patterns to locate the exact faces or features the request names before you change anything. Confirm you have identified the correct features (count, location, orientation) against the reference renders.
2. Apply ONLY the requested change. Work from the named features and the stated dimension (e.g. a 6 mm offset), not by eye.
3. Checkpoint: call validate(); as soon as it PASSES, export() AND save_snapshot() — a valid edited solid is now both on disk and recoverable in-session. If validate() FAILs, fix and re-validate before exporting.
4. Iterate to fidelity (several passes): render_view() and compare against the reference renders — confirm the requested change is present and correct everywhere it should be, and that nothing else moved. Use measure() to verify the edit dimension. Fix the largest discrepancy with execute(), then validate() again; only when it PASSES, export() and save_snapshot(). Repeat until the edit is correct and complete.

## Rules (read carefully)
- Validity is judged ONLY by validate(). measure() (dimensions, counts) and render_view() (appearance) do NOT reveal whether the solid is watertight and manifold — never infer validity from them or from a sensible-looking volume.
- After every geometry-changing execute(), call validate() before you render, measure, or export. Never build a new change on top of a solid you have not just re-validated.
- If a fix makes validate() FAIL, repairing it is the next action — or restore_snapshot() back to the last valid state — before anything else. Never keep iterating on an invalid solid.
- Never export a solid that did not just PASS validate(), so you never overwrite a good checkpoint with an invalid one. Always leave a valid export on disk.
- Change only what the request asks. Do not re-model untouched regions, and do not add or remove features the request does not mention.
