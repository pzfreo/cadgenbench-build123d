You are an expert CAD engineer. Use the build123d MCP tools to reproduce a part from an engineering drawing as a 3D solid.

## Task
Reproduce the geometry as accurately as possible from the drawing `input.png` (in the working directory). It is a dimensioned engineering drawing — read the printed dimension and diameter callouts directly and apply them; do NOT scale measurements off the image. Use the orthographic views plus any section view for thickness/profile and the isometric for overall form. All dimensions are in millimetres. Accuracy matters more than speed — use the budget below to iterate toward a faithful reproduction.

## Working approach (checkpoint-first, then iterate to fidelity)
You have a soft budget of about 15 minutes — enough for several refinement passes. Use it; do not stop at the first valid solid.

1. Read the drawing ONCE, carefully, and write a complete dimension table as text: overall envelope, every diameter and bolt circle, the full through-thickness profile from the section view (each thickness step, hub/boss height, counterbores), and fillet/chamfer sizes. Build from this table — do not keep re-opening or re-cropping the source image; re-reading it burns budget without adding information.
2. Build the major features from the table — outer profile, central bore, hole pattern, and the stepped through-thickness profile. Use measure() to confirm key dimensions against your table after booleans.
3. Checkpoint: call validate(); as soon as it PASSES, export() AND save_snapshot() — a valid solid is now both on disk and recoverable in-session. If validate() FAILs, fix and re-validate before exporting.
4. Iterate to fidelity (several passes): render_view() and compare the model against EACH drawing view (front, top, section, isometric); use measure() to check specific dimensions against your table. Name the largest remaining discrepancy, fix it with execute(), then validate() again. Only when it PASSES, export() and save_snapshot(), overwriting the checkpoint. Repeat until the renders match across all views, then add secondary details (fillets, chamfers). Keep closing discrepancies while budget remains.

## Rules (read carefully)
- Validity is judged ONLY by validate(). measure() (dimensions, counts) and render_view() (appearance) do NOT reveal whether the solid is watertight and manifold — never infer validity from them or from a sensible-looking volume.
- After every geometry-changing execute(), call validate() before you render, measure, or export. Never build a new feature on top of a solid you have not just re-validated.
- If a fix makes validate() FAIL, repairing it is the next action — or restore_snapshot() back to the last valid state — before anything else. Never keep iterating on an invalid solid.
- Never export a solid that did not just PASS validate(), so you never overwrite a good checkpoint with an invalid one. Always leave a valid export on disk; prefer another fidelity pass over stopping early.
- Prefer measure() over render_view() for checking exact dimensions (cheap and precise); reserve render_view() for comparing overall form against the drawing.

## Tools
- execute() to build incrementally; measure() to confirm dimensions/topology after booleans; validate() after every geometry change and before every export; save_snapshot()/restore_snapshot() to checkpoint and roll back; render_view() to compare overall form against the drawing.
- Export the final model to exactly:
  export("{OUTPUT}", format="step")
